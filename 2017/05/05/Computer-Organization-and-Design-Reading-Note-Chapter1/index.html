Computer Organization and Design Chapter1

Posted on 2017-05-05

名词解释

  1. throughput 吞吐率 = 带宽（bandwidth）

    性能的度量参数，表示单位时间内完成的任务数量

  2. response time 响应时间 = 执行时间（excution time）

    完成任务所需总时间，包括硬盘访问，内存访问，I/O活动，操作系统开销和CPU执行时间等。

  3. CPU execution time

    执行某一任务只在CPU上花费的实践，不包括I/O或运行其它程序的时间。

    3.1. user CPU time = CPU performance
    程序本省所花费的CPU时间

    3.2. system CPU time = system performance
    操作系统为用户服务（执行程序）花去的在操作系统上的CPU时间

  4. clock cycle 时钟周期

    计算机一个时钟周期的时间

  5. 时钟周期数

    CPU执行时间内一共需要多少时钟周期

  6. CPI(clock cycle per instruction)

    执行每条指令的平均时钟周期数

  7. IPC(instruction per clock cycle)

    CPI的倒数

  8. Amdahl定律

    改进大小与总性能提高的公式，可能存在无法改进的情况。例：p33

  9. MIPS(million instructions per second)

    基于百万条指令的程序执行速度的一种测量，MIPS有三个缺点

公式

  1. 性能=1/执行时间
  2. 对一个程序来说：CPU执行时间 = CPU时钟周期数 * 时钟周期时间 (1/时钟频率)
  3. 时钟周期数 = 程序指令数 * 每条指令平均时钟周期数CPI
  4. 整合公式：CPU执行时间 = 指令数 CPI 时钟周期时间
  5. Amdahl定律：改进后的执行时间=受改进影响的执行时间/改进量+不受影响的执行时间
  6. MIPS = 指令数/(执行时间$$10^6$$) = 时钟频率/(CPI $$10^6$$)

Remark

  1. 性能的度量针对的是CPU的执行时间,执行时间是唯一有效且不可推翻的度量方法
  2. 一般降低响应时间都可以增加吞吐率。
  3. GHz=$$10^9$$
